应用程序框架基础

应用程序框架概述

  • 用户: 应用内的交互，应用间的交互，应用的跨设备流转

  • 开发者: 提供应用进程管理，应用生命周期调度，应用组件生命周期调度，应用上下文环境，系统环境监听等相关能力

  • 应用程序框架
    是一种编程框架，用来简化应用程序的开发过程，提高代码的可重用性和可维护性。可以帮助开发人员更快速、更高效地开发应用程序

  • 应用模型
    应用模型是一个应用程序的模型，它是一种抽象描述。用于表示应用程序的不同方面，例如应用组件、进程模型、线程模型、任务模型以及包管理。
    从模型演进角度分为FA模型以及Stage模型，目前长期演进策略是Stage模型

两者之间的关系：
应用程序框架可以被看做是应用模型的一种方式。开发人员可以根据应用模型描述应用程序的结构和行为，然后使用应用程序框架来实现这些描述。

Stage应用模型设计思想

  • 为复杂应用而设计

    • 不同应用组件间实例共享
    • 基于面向对象的开发方式

  • 原生支持应用组件级的跨端迁移和多端协同

    • UI Ability与UI分离
    • UI展示与服务能力合一的UI Ability组件

  • 支持多设备和多窗口形态

    • UIAbility生命周期定义
    • 组件管理和窗口管理解耦

  • 平衡应用能力与系统管控成本

    • 严格的后台管控
    • 基于场景的服务机制
    • 严格进程模型

工具目录结构介绍

  • 文件类型
    • 资源文件
    • 配置文件
    • ArkTS文件

UIAbility组件基础

  • UIAbility生命周期
    UIAbility Start
    Create
    WindowStageCreate
    Foreground
    Background
    WindowStageDestroy
    Destroy
    UIAbility End

  • UIAbility是一种包含用户界面的应用组件，用于和用户进行交互，UIAbility是系统调度的单元，提供窗口用于界面绘制

  • UIAbility的创建和对应页面的创建

  • UIAbility内页面的跳转

  • UIAbility的创建、前后台切换、销毁的生命周期状态
